Introduction

P-Pay API exposes endpoints of our for B2B, AND B2C that you will require to invoke as you integrate with your organization . To invoke the API, you can utilize tools such as Postman REST client, curl, and HTTPie. In this documentation, we have included sample code snippets in various languages along with the expected JSON responses.

P-Pay API implements the JSON API Specification

Getting Started

To start integrating with P-Pay API, follow the steps below

1.Register as a developer

You need an active P-pay Account.

Navigate to the registration page and complete registration process

2. Create an application

. Login to developer portal with your P-Pay developer account and navigate to Dashboard

. Under the DASHBOARD menu, select My Applications

. Make sure you are on the Sandbox tab to create a sandbox application to obtain the API credentials you'll use while you're developing code.

. After you test before you go live, switch to the Live tab to create a Production application and get live credentials and guidance.

3. Access resources

To access resources, you need an Access Token can be obtained by posting the Client ID and Secret generated when you created your application to the end point as shown in the Authentication section. Remember to store your credentials securely.
